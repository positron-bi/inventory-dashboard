# داشبورد موجودی انبار — گروه پوزیترون

این پوشه برای انتشار روی **GitHub Pages** آماده است و فقط شامل فایل‌های امن می‌باشد.

## 🔒 محافظت با رمز عبور

فایل `index.html` یک داشبورد **رمزنگاری‌شده** است. کل داده‌ها با الگوریتم
**AES-256-GCM** و کلیدِ مشتق‌شده از رمز عبور با **PBKDF2-SHA256 (۲۰۰٬۰۰۰ تکرار)**
رمزنگاری شده‌اند. رمزگشایی فقط داخل مرورگر و با Web Crypto انجام می‌شود
(کاملاً آفلاین، بدون نیاز به اینترنت یا CDN).

- بدون رمز درست، حتی از سورس صفحه هم هیچ داده‌ای دیده نمی‌شود.
- رمز عبور هیچ‌جا داخل فایل ذخیره نمی‌شود؛ فقط برای رمزگشایی استفاده می‌شود.

> ⚠️ **نکته‌ی امنیتی:** چون فایل عمومی است، هرکسی با داشتن رمز می‌تواند داده‌ها را
> ببیند. برای تغییر رمز، داشبورد را دوباره با رمز جدید بسازید (پایین را ببینید).
> رمز عبور را در issue/commit ننویسید.

## 🚀 انتشار روی GitHub Pages

۱. یک ریپازیتوری بسازید و **فقط محتوای همین پوشه (`site/`)** را در آن قرار دهید.
۲. در GitHub: `Settings → Pages → Branch: main → /(root) → Save`.
۳. چند دقیقه بعد، داشبورد روی آدرس `https://<username>.github.io/<repo>/` در دسترس است.

برای انتشار سریع با خط فرمان:

```bash
cd site
git init
git add .
git commit -m "Encrypted inventory dashboard"
git branch -M main
git remote add origin https://github.com/<username>/<repo>.git
git push -u origin main
```

## 🔁 ساخت دوباره با داده‌ی جدید یا رمز جدید

داشبورد رمزنگاری‌شده با اسکریپت اصلی ساخته می‌شود:

```bash
# با آرگومان رمز:
python generate_dashboard.py --password "<رمز شما>"

# یا با متغیر محیطی:
set DASHBOARD_PASSWORD=<رمز شما>   &  python generate_dashboard.py
```

سپس فایل خروجی `dashboard_موجودی.html` را به‌نام `index.html` در این پوشه جایگزین کنید.

> اسکریپت‌های `generate_dashboard.py` و `جامع انبار.py` و فایل‌های اکسل **نباید**
> روی گیت‌هاب منتشر شوند (حاوی نام کاربری/رمز سامانه‌ها و داده‌ی خام هستند).
